The 3 V's of Big Data

Core dimensions that define big data challenges — and the AWS services that address them
BIG DATA

📦 VOLUME

How much data?
  • Scale: GBs → TBs → PBs
  • Logs, transactions, sensors
  • Social media, clickstreams
  • Growing exponentially over time
AWS → S3, Redshift, Glue, EMR, Athena

⚡ VELOCITY

How fast does it arrive?
  • Batch: hourly / daily
  • Near real-time: minutes
  • Real-time: sub-second
  • Streaming vs scheduled ingestion
AWS → Kinesis Streams, Firehose, MSK, Lambda

🔀 VARIETY

What types of data?
  • Structured: tables, CSV, RDS
  • Semi-structured: JSON, XML, logs
  • Unstructured: images, video, free text
  • Schema evolution and format conversion
AWS → S3 (all types), Glue Data Catalog, OpenSearch
