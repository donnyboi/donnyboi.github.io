Ensemble Methods

Bagging, Boosting, Stacking — what they do, when to use them, how they differ

The core idea

A single model has limits. Ensemble methods combine multiple models to produce predictions that are more accurate and more robust than any individual model could be. There are three main flavours — each solves a different problem.

The three families

🌳 Bagging — Bootstrap Aggregating

Parallel democracy of clones

Train many copies of the same model on different random subsets of the data (called bootstrap samples). Each model votes, and you take the average or majority vote.

Goal: reduce variance (overfitting). If one model overfits a weird quirk in its subset, the others will outvote it.

Famous example: Random Forest = bagging applied to decision trees.

🚀 Boosting — Sequential Error Correction

Team of specialists fixing each other's mistakes

Train models one at a time, sequentially. Each new model focuses on what previous models got wrong. Final prediction is a weighted sum of all models.

Goal: reduce bias (underfitting). Each model targets the remaining errors until the ensemble fits the data well.

Famous examples: XGBoost, LightGBM, AdaBoost, Gradient Boosting.

🏛️ Stacking — Meta-Learning

Expert panel with a judge

Train different types of models (e.g., XGBoost + Neural Net + Logistic Regression) on the same data. Feed their predictions as features into a meta-model that learns how to best combine them.

Goal: capture diverse strengths. Different algorithms excel at different patterns; stacking blends their strengths.

Famous example: How most Kaggle competitions get won.

Mental models

🗳️
Bagging — Election
Hundreds of identical voters, each hearing a slightly different version of the facts. Majority rules. One misinformed voter gets drowned out.
🏥
Boosting — Medical Specialist Chain
GP diagnoses → cardiologist refines → surgeon handles what they couldn't. Each specialist focuses on what the previous couldn't solve. Errors accumulate into corrections.
⚖️
Stacking — Expert Panel + Judge
A doctor, an engineer, and a lawyer each give their verdict on a complex case. A judge listens to all three and makes the final call based on who tends to be right in this kind of situation.

When to use which

"Data is noisy and my single model is overfitting"
→ Use Bagging (Random Forest) to reduce variance
"Tabular data classification, I want max accuracy"
→ Use Boosting (XGBoost) — dominant on structured data
"My best single model is XGBoost but I need a bit more performance"
→ Try Stacking with XGBoost + a neural net + linear model
"I need interpretable predictions"
→ Prefer Bagging (feature importances work naturally; avoid stacking)
"I have millions of rows and need fast training"
→ Bagging or LightGBM (boosting variant designed for scale)
"Small dataset, underfitting"
→ Boosting — it reduces bias, which is what you need

Key XGBoost hyperparameters (exam-relevant)

num_round — number of boosting rounds (trees). More = better fit but risk overfitting.

eta — learning rate. Lower means slower but more stable training. Typical 0.01–0.3.

max_depth — tree depth. Deeper captures more interactions but overfits.

subsample — fraction of rows per tree (adds randomness like bagging).

scale_pos_weight — handles class imbalance.

💡 Memory hook

  • Bagging → Bootstrap (random samples, parallel)
  • Boosting → Builds on mistakes (sequential)
  • Stacking → Specialists + judge (diverse + meta)
🎯 Exam trigger words

"Reduce overfitting" → Bagging / Random Forest  ·  "Max accuracy on tabular" → XGBoost  ·  "Combine multiple model types" → Stacking  ·  "Imbalanced classification" → XGBoost + scale_pos_weight
